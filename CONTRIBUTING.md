# Contributing to InfoShield

Thank you for your interest in contributing to InfoShield. The goal is to keep this project accessible, educational, and easy to maintain.

## How to contribute

1. Fork the repository.
2. Create a new branch for your change:
   ```bash
git checkout -b feature/your-feature-name
```
3. Install dependencies and run the app locally.
4. Make your changes.
5. Submit a pull request with a clear summary of your updates.

## Development workflow

- Keep changes small and focused.
- Use clear commit messages.
- Rebuild the app before submitting changes:
  ```bash
npm run build
```

## Code style

- Use React functional components.
- Prefer Tailwind CSS utility classes for styling.
- Keep component logic clean and reusable.
- Avoid duplicated code when possible.

## Testing changes

- Run the application locally with `npm run dev`.
- Verify the UI and navigation flow in the browser.
- Check that the build succeeds with `npm run build`.

## Reporting issues

If you find a bug or have an improvement idea, please open an issue with:

- steps to reproduce
- expected behavior
- actual behavior
- relevant screenshots or console logs
